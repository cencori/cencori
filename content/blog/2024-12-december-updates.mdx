## TypeScript SDK: First-Class Developer Experience

We published the official Cencori TypeScript SDK to npm. Instead of writing raw fetch calls, you can now use a typed client:

```bash
npm install cencori
```

```typescript
import { Cencori } from 'cencori';

const cencori = new Cencori({
  apiKey: process.env.CENCORI_API_KEY!,
});

// Non-streaming
const response = await cencori.ai.chat({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
});

console.log(response.content);

// Streaming
const stream = cencori.ai.chatStream({
  model: 'claude-3-opus',
  messages: [{ role: 'user', content: 'Tell me a story' }],
});

for await (const chunk of stream) {
  process.stdout.write(chunk.delta);
}
```

**Why it matters:** Type safety catches errors at compile time. Streaming support enables real-time chat UIs. Error classes like `SafetyError` and `RateLimitError` let you handle edge cases gracefully.

## Python SDK: For Python Teams

A potential customer's backend was in Python. We built the SDK they needed:

```bash
pip install cencori
```

```python
from cencori import Cencori

cencori = Cencori(api_key="your-api-key")

# Non-streaming
response = cencori.ai.chat(
    messages=[{"role": "user", "content": "Hello!"}],
    model="gpt-4o",
)

print(response.content)

# Streaming
for chunk in cencori.ai.chat_stream(
    messages=[{"role": "user", "content": "Tell me a story"}],
    model="claude-3-opus",
):
    print(chunk.delta, end="")
```

Full feature parity with the TypeScript SDK. Built with `httpx` for async support and type-safe dataclasses for all responses.

**Published to PyPI:** [pypi.org/project/cencori](https://pypi.org/project/cencori/)

## Cenpact Design System

We redesigned Cencori with a new design language we call **Cenpact** (Cencori + Compact).

### The Philosophy

| Principle | What It Means |
|-----------|---------------|
| **Dense but Breathable** | Pack information tightly while maintaining visual hierarchy |
| **Subtle Sophistication** | Muted colors, soft borders, understated hover effects |
| **Developer-First** | Design for power users who value efficiency |
| **Dark Mode Native** | Built for dark mode first |

### What Changed

- **Dashboard:** Compact cards, smaller typography, tighter spacing
- **Blog:** Simplified cards, denser content, cleaner article pages
- **Docs:** Reduced padding, smaller headers, focused reading experience
- **Dynamic OG Images:** Auto-generated social preview cards with Cenpact styling

The result is a UI that shows more information with less scrolling, without feeling cramped.
