Modern software development runs on AI agent loops. Tools like Cursor, Cline, Continue, and Zed spend your entire workday firing requests — scanning repositories, running test matrices, refactoring large codebases. Managing separate API subscriptions for Anthropic, OpenAI, and DeepSeek while watching unpredictable per-token bills spike mid-workflow is a real problem.

**Cencori Code** fixes it. It is a secure, high-throughput subscription proxy built specifically for power-user engineers. You get one `csk_` gateway token that unlocks a curated catalog of frontier models. Billing is handled through a simple internal credit system — no raw token costs exposed to you, no financial wallet juggling.

## How it works

Cencori Code is fully OpenAI-compatible. You do not install a new SDK. You change two values in your IDE agent settings:

| Setting | Value |
|---------|-------|
| Base URL | `https://api.cencori.com/v1` |
| API Key | Your `csk_` gateway token |

Your IDE then routes all model requests through the Cencori Code proxy. The proxy authenticates you against your active subscription tier, estimates the worst-case credit cost of the request, locks that reservation atomically, forwards the stream to the upstream provider, and settles the actual credit cost once the stream completes — all in under 30ms of added overhead.

## Credits: predictable billing

Usage is measured in **Credits** rather than raw tokens. One Credit equals $0.001. This abstraction lets Cencori adjust upstream margins behind the scenes without changing your subscription plan.

Every inbound request goes through a **three-tier rolling window** before it is dispatched:

| Window | Purpose |
|--------|---------|
| 5-Hour Rolling | Catches and stops runaway agent loops before they cause financial damage |
| 7-Day Rolling | Manages sustained delivery across an engineering week |
| Monthly Billing | Hard cap aligned to your Paystack renewal date |

If any single window is exhausted, the request is blocked immediately with an HTTP `429` response that includes the specific window that was depleted and its reset time.

---

## Key management

You can generate up to **5 named gateway API keys** — one per workflow or machine (e.g., Laptop, Desktop, Remote Server). Keys are masked after creation. The dashboard shows a `last_used_at` timestamp for each key so you can audit live connectivity.
