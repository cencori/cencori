Cencori Code is an OpenAI-compatible proxy. You do not install anything — you point your existing IDE agent at a new base URL and swap in your `csk_` token.

<Steps>
  <Step>
    <StepTitle>Subscribe and generate a key</StepTitle>
    <StepContent>
      <StepDescription>
        Sign up at [cencori.com/signup](/signup) and choose a subscription tier. Once your account is active, open **Dashboard → API Keys** and click **Generate New Key**. Give it a name (e.g. `Laptop` or `Cursor`) so you can track it later.

        Copy the key immediately — it is only shown once.
      </StepDescription>

      ```bash
      # Store it in your shell or IDE's secrets manager
      CENCORI_API_KEY=csk_...
      ```
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Configure your IDE agent</StepTitle>
    <StepContent>
      <StepDescription>
        Pick your tool below. All you are changing is the **base URL** and **API key**.
      </StepDescription>

      ### Cursor

      Open **Cursor → Settings → Models → OpenAI API Key** and set:

      ```
      Base URL:  https://api.cencori.com/v1
      API Key:   csk_...
      ```

      Then add the models you want (`claude-sonnet-4.5`, `gpt-4o`, `deepseek-v3`) to your model list.

      ### Cline (VS Code)

      Open **Cline Extension Settings → API Provider** and select **OpenAI Compatible**, then set:

      ```
      Base URL:  https://api.cencori.com/v1
      API Key:   csk_...
      ```

      ### Continue

      Edit `.continue/config.json` in your project root:

      ```json
      {
        "models": [
          {
            "title": "Claude Sonnet 4.5 (Cencori)",
            "provider": "openai",
            "model": "claude-sonnet-4.5",
            "apiBase": "https://api.cencori.com/v1",
            "apiKey": "csk_..."
          },
          {
            "title": "GPT-4o (Cencori)",
            "provider": "openai",
            "model": "gpt-4o",
            "apiBase": "https://api.cencori.com/v1",
            "apiKey": "csk_..."
          }
        ]
      }
      ```

      ### Zed

      Edit your Zed `settings.json`:

      ```json
      {
        "language_models": {
          "openai": {
            "api_url": "https://api.cencori.com/v1",
            "available_models": [
              { "name": "claude-sonnet-4.5", "max_tokens": 8096 },
              { "name": "gpt-4o", "max_tokens": 8096 },
              { "name": "deepseek-v3", "max_tokens": 8096 }
            ]
          }
        }
      }
      ```

      Set the API key via Zed's **AI Provider** prompt when it appears.
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Verify with cURL</StepTitle>
    <StepContent>
      <StepDescription>
        Run this directly to confirm your key and the endpoint are working, with no IDE layer in between.
      </StepDescription>

      ```bash
      curl https://api.cencori.com/v1/chat/completions \
        -H "Authorization: Bearer $CENCORI_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "model": "gpt-4o",
          "messages": [{"role": "user", "content": "Reply with OK."}],
          "max_tokens": 10
        }'
      ```

      A successful response returns a standard OpenAI-shaped JSON body. Your credit balance decreases by the actual cost of the request after the stream settles.
    </StepContent>
  </Step>
</Steps>

## Supported models

| Model ID | Provider | Best for |
|----------|----------|----------|
| `claude-sonnet-4.5` | Anthropic | Complex code reasoning, large context |
| `gpt-4o` | OpenAI | General-purpose, strong all-rounder |
| `deepseek-v3` | DeepSeek | High-volume, cost-efficient tasks |

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `401 Unauthorized` | Key is invalid or not yet active | Check the key was copied fully; regenerate if needed |
| `429 Rate Limited` | A rolling window is exhausted | Check which window in the error body; wait for reset or check your plan |
| Model not found | Model ID typo or not on your tier | Check [supported models](/docs/ai/models) for exact IDs |
| Request appears in no logs | IDE is still hitting the provider directly | Confirm base URL is set to `https://api.cencori.com/v1` and not the provider's URL |
