Cencori Code is an OpenAI-compatible proxy. For most IDE agents, there is nothing to install — you set a base URL and an API key. If you are building a backend service that calls the API directly, you can use any OpenAI-compatible SDK or plain HTTP.

## IDE Agents

<Steps>
  <Step>
    <StepTitle>Get your gateway key</StepTitle>
    <StepContent>
      <StepDescription>
        Open **Dashboard → API Keys**, click **Generate New Key**, and give it a name. You can create up to 5 keys — one per workflow or machine.
      </StepDescription>

      ```bash
      CENCORI_API_KEY=csk_...
      ```
    </StepContent>
  </Step>

  <Step>
    <StepTitle>Set base URL and key in your tool</StepTitle>
    <StepContent>
      <StepDescription>
        Every tool below uses the same two values.
      </StepDescription>

      | Setting | Value |
      |---------|-------|
      | Base URL | `https://api.cencori.com/v1` |
      | API Key | `csk_...` |

      **Cursor** — Settings → Models → Add model → set base URL and key under OpenAI API.

      **Cline** — Extension Settings → API Provider: `OpenAI Compatible` → base URL and key.

      **Continue** — `.continue/config.json`:

      ```json
      {
        "models": [{
          "provider": "openai",
          "model": "claude-sonnet-4.5",
          "apiBase": "https://api.cencori.com/v1",
          "apiKey": "csk_..."
        }]
      }
      ```

      **Zed** — `settings.json` under `language_models.openai`:

      ```json
      {
        "language_models": {
          "openai": {
            "api_url": "https://api.cencori.com/v1"
          }
        }
      }
      ```
    </StepContent>
  </Step>
</Steps>

## Programmatic Use

### OpenAI Node SDK

```typescript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.cencori.com/v1",
  apiKey: process.env.CENCORI_API_KEY,
});

const response = await client.chat.completions.create({
  model: "claude-sonnet-4.5",
  messages: [{ role: "user", content: "Hello" }],
});
```

### OpenAI Python SDK

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://api.cencori.com/v1",
    api_key=os.environ["CENCORI_API_KEY"],
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello"}],
)
```

### cURL

```bash
curl https://api.cencori.com/v1/chat/completions \
  -H "Authorization: Bearer $CENCORI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v3",
    "messages": [{"role": "user", "content": "Hello"}],
    "max_tokens": 256
  }'
```

## Notes

- The base URL must end at `/v1`. The SDK appends `/chat/completions` automatically.
- Use `Authorization: Bearer csk_...` for all requests to `api.cencori.com/v1`.
- Keep `csk_` keys on the server. Do not expose them in browser bundles or `NEXT_PUBLIC_*` env vars.
- Audio, image generation, and embeddings endpoints are not available in the initial release. Text-based chat completions only.
