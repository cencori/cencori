Prompt Cache reduces AI costs by **30-60%** and latency by **95%** by caching LLM responses. Supports both exact and semantic matching with full analytics.

## How It Works

### Two-Tier Caching

| Layer | Speed | Use Case |
|-------|-------|----------|
| **Exact Match** (Redis) | <10ms | Identical prompts |
| **Semantic Match** (pgvector) | ~50ms | Similar prompts (embedding similarity) |

1. **Request arrives** → Check exact cache (Redis)
2. **No exact hit?** → Check semantic cache (pgvector)
3. **No semantic hit?** → Call provider, store response

### Cache Key

The cache key includes:
- `project_id`
- `model`
- `messages` (normalized, whitespace-trimmed)
- `temperature` (normalized within `max_cacheable_temperature`)

## Features

### Exact Match
- **Instant hits** - Redis-backed, <10ms latency
- **SHA-256 hash** of normalized prompt
- Case-insensitive, whitespace-normalized

### Semantic Match  
- **Similarity search** using text-embedding-004
- **Configurable threshold** (default: 0.95)
- Best for paraphrased or varied prompts

### Smart Temperature Handling
Only caches low-temperature (deterministic) responses:
- `max_cacheable_temperature` defaults to `0.2`
- Higher temps → skipped from cache
- Preserves creative outputs

### Model Exclusion
Exclude specific models from caching:
- Reasoning models (`o3-mini`, `o1`, etc.)
- Fine-tuned models
- Expensive models

## Configuration

### Dashboard

Navigate to **Project → Cache → Settings**:

```json
{
  "cache_enabled": true,
  "exact_match_enabled": true,
  "semantic_match_enabled": true,
  "ttl_seconds": 3600,
  "similarity_threshold": 0.95,
  "max_entries": 10000,
  "excluded_models": ["o3-mini", "o1"],
  "max_cacheable_temperature": 0.2
}
```

### API

Enable/disable per request:

```typescript
import { Cencori } from 'cencori';

const cencori = new Cencori({ apiKey: 'ck_...' });

const response = await cencori.chat({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Explain quantum entanglement' }],
  // Override project settings
  cache: {
    enabled: true,
    mode: 'semantic'
  }
});
```

### Headers

```bash
curl https://api.cencori.com/v1/chat/completions \
```

## Examples

### Exact Match (Recommended for Production)

```typescript
const response = await cencori.chat({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Classify sentiment: ${text}' }],
  cache: { mode: 'exact' }
});
```

### Semantic Match (Best for Cost Savings)

```typescript
const response = await cencori.chat({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Summarize this article: ${article}' }],
  cache: { 
    mode: 'semantic',
    threshold: 0.95  // stricter = more accurate
  }
});
```

## Best Practices

1. **Enable Exact Match first** - safest, fastest
2. **Start with high threshold** (0.95+) for semantic
3. **Exclude reasoning models** (`o3-mini`, `o1`, `claude-3-opus`)
4. **Lower TTL for fast-changing data** (1 hour)
5. **Higher TTL for static content** (24 hours)

## Cache Invalidation

### Manual (Dashboard)

Go to **Project → Cache → Analytics → Clear All**

### API

```bash
# Clear specific cache key
curl -X DELETE https://api.cencori.com/v1/cache \
  -H "Authorization: Bearer sk_..." \
  -H "Content-Type: application/json" \
  -d '{"cache_key": "abc123"}'

# Clear all
curl -X DELETE https://api.cencori.com/v1/cache \
  -H "Authorization: Bearer sk_..." \
  -d '{"all": true}'

# Clear by model
curl -X DELETE https://api.cencori.com/v1/cache \
  -H "Authorization: Bearer sk_..." \
  -d '{"model": "gpt-4o-mini"}'
```
