# Shipping Report: March 2, 2026 to April 1, 2026

## Scope

- Branch reviewed: `master`
- Window reviewed: `2026-03-02` through `2026-04-01`
- Commits reviewed: `94`
- Primary author: `Bola Banjo <omogbolahanng@gmail.com>`

This report is meant to answer one question clearly: what shipped in the last 30 days, how it grouped into product work, and which commits support each shipped area.

## Executive Summary

Over this 30-day window, Cencori shipped meaningful work across the AI gateway, Scan, observability, enterprise controls, internal operations tooling, billing, edge integrations, and deployment reliability.

The highest-signal outcomes were:

- The AI gateway expanded to support custom provider routing, GPT-5.3 Instant, GPT-5.4, GPT-5.4 Pro, semantic caching, and a later reliability hardening pass that made Redis and semantic cache failures fail soft instead of taking chat and embeddings down.
- Cencori Scan matured rapidly: repo-aware chat, continuity memory, AI quality review, investigation stream improvements, stale-run recovery, persistent diff/PR actions, and launch content all landed in the same period.
- Observability became materially stronger with a redesigned intelligence panel, anomaly alerting, unified HTTP traffic views, platform-wide event tracking, a custom geo map, configurable circuit breaker thresholds, and organization-level audit logs with export/backfill support.
- Internal operations shipped a real admin surface: shared `/internal` shell, sign-up and access flows, broadcast user emails, and a branded internal email system with WYSIWYG compose, sender profiles, avatars, image uploads, dark mode, and send history.
- Commercial and admin capabilities advanced with end-user billing, Stripe Connect invoicing flows, usage/billing docs, internal task management, SSO/SAML, model mappings, and usage export.
- Deployment and release ops were actively maintained through TypeScript/build fixes, a build recovery around fonts and UI peer dependencies, Vercel Hobby cron remediation, and a GitHub Actions-based replacement for unsupported scheduled jobs.

## Shipped Work By Area

### 1. AI Gateway, Models, and Reliability

This month materially expanded the gateway and then tightened its operational behavior after production issues surfaced.

- Custom provider endpoint routing shipped in `7d57e20` on March 4, giving the gateway support for user-configured custom upstream endpoints.
- GPT-5.3 Instant support shipped across model catalogs in `e0adfee` on March 4, with the launch content in `4c4900e`.
- Semantic caching for `/api/ai/chat` shipped in `bbf4ddf` on March 5, followed by environment and documentation alignment in `ea7fd3d`, `0da5161`, `370fba8`, and `4d41e0a`.
- GPT-5.4 and GPT-5.4 Pro shipped platform-wide in `6dcdaa3` on March 5, with release content in `09281b3` and `7ba8721`.
- Prompt registry and cache-related UI work landed in `ccaa6b2` on March 14, extending the operational surface around caching and prompt management.
- Gateway reliability was significantly hardened in `6a10831` on April 1. This patch made Redis-backed rate limiting fail open, softened semantic cache failures, dimension-validated embeddings before cache writes, and improved provider-vs-infra error attribution.
- The April 1 deployment recovery also fixed production build blockers in `6945b90`, including replacing remote Google font fetching with bundled `geist` fonts and restoring a missing `antd` peer dependency required by UI packages.

Why this matters:

- Customers gained more model choice and more flexible upstream routing.
- Caching moved from concept to live gateway capability.
- The gateway became safer under partial infra failure, which is critical for real production traffic.
